## 速览：Pi 是什么？

Pi 是一个**终端 AI 编码智能体**（Terminal AI Coding Agent）。它的核心理念：

```
┌────────────────────────────────────────────────────────────┐
│                        终端（Terminal）                       │
│                                                            │
│   ┌─────────┐    ┌──────────────────┐    ┌──────────────┐  │
│   │  用户    │◄──►│   TUI 界面        │◄──►│  Agent 循环   │  │
│   │  输入    │    │  (编辑器 / 渲染)   │    │              │  │
│   └─────────┘    └──────────────────┘    └──────┬───────┘  │
│                                                  │          │
│   项目信任 / 认证 / 模型解析 / 扩展 / Skill / 工具  │          │
│                                       ┌──────────▼───────┐  │
│                                       │  LLM 提供商       │  │
│                                       │ (OpenAI / Anthropic│  │
│                                       │  / 30+ 自定义)     │  │
│                                       └──────────┬───────┘  │
│                                                  │          │
│                                       ┌──────────▼───────┐  │
│                                       │  工具执行         │  │
│                                       │ read / write /   │  │
│                                       │ bash / edit / ...│  │
│                                       └──────────────────┘  │
└────────────────────────────────────────────────────────────┘
```

### 四个核心包（v0.79.10）

| 包名                  | 职责                                                          | 规模（非测试 `.ts`）                        | 类比           |
| --------------------- | ------------------------------------------------------------- | ------------------------------------------- | -------------- |
| **`pi-ai`**           | 统一 LLM API：Models 运行时、Provider factory、API 实现、认证 | ~144 个文件，含大量生成的 Provider 模型目录 | "数据库驱动层" |
| **`pi-agent-core`**   | 智能体运行时：`AgentLoop`、工具调用、事件流、`AgentHarness`   | ~25 个文件                                  | "ORM / 事务层" |
| **`pi-coding-agent`** | CLI 入口、项目信任、扩展、会话管理、TUI 集成、工具实现        | ~157 个文件                                 | "应用层"       |
| **`pi-tui`**          | 独立可复用的终端 UI 库                                        | ~28 个文件                                  | "UI 框架"      |

> 注意：`pi-ai` 和 `pi-coding-agent` 的文件数看起来很多，其中相当一部分是**按 Provider 生成的模型目录**（`providers/*.models.ts`）和工具实现，手写核心逻辑仍然集中在几千行内。

### 核心数据流

```
用户输入
  → AgentSession.prompt(text)            // 消息预处理、扩展拦截、skill 展开
  → Agent.prompt(messages)               // 启动 prompt
  → runAgentLoop(messages, context)      // ★ 核心循环
      → streamAssistantResponse()        // 通过 pi-ai Models 调用 LLM
      → executeToolCalls()               // 执行工具调用
      → 将 tool result 写入 context
      → 循环直到 LLM 不再调用工具
```

与旧版最大的区别：从 v0.79 开始，LLM 调用不再经过旧的全局 `stream()` / `complete()`，而是通过 **Models 运行时** (`createModels()` + Provider factory) 完成，认证也统一走 `CredentialStore` / `AuthContext`。

## 为什么读这份文档？

读完本系列，你将：

1. **理解一个生产级编码智能体如何工作** — 不只是 demo，而是处理了边缘情况、错误恢复、性能优化、项目安全的真实产品
2. **掌握 LLM Agent 的核心模式** — Provider 抽象、Models 运行时、事件驱动、流式处理、工具调用循环
3. **学会终端 UI 的构建方法** — 差分渲染、ANSI 转义、Kitty 协议、IME 支持
4. **理解 v0.79 的新架构** — 项目信任、`createModels()`、`api/` 与 `providers/` 分离、认证体系
5. **能够独立扩展 Pi** — 写 Extension、Skill、Tool、Theme、自定义 Provider

## 快速开始

```bash
# 1. 克隆源码（注意仓库已从 pi-mono 迁移到 pi）
git clone https://github.com/earendil-works/pi.git
cd pi

# 2. 安装依赖（不执行生命周期脚本）
npm install --ignore-scripts

# 3. 检查代码质量（类型 + lint + 依赖检查）
npm run check

# 4. 直接运行源码（无需构建）
npx tsx packages/coding-agent/src/cli.ts --help
```

> Node.js 版本要求：`>= 22.19.0`。Pi 使用 TypeScript 的 `tsgo` 原生编译预览版进行构建，开发时可直接通过 `tsx` 运行源码。
